# 账户调研 Skill 使用指南

---

## 这份文档是什么

这是一份操作手册，说明如何使用已安装的 `account-research` Skill 通过 AI 完成账户类型调研。

---

## 你需要准备什么

| 需要 | 说明 |
|------|------|
| Claude Code | 已安装并可正常使用 |
| 一个调研目标 | 例如：新加坡的联名账户，或澳大利亚的退休账户 |

---

## 核心概念（30 秒读懂）

`account-research` 是一个已安装在你的 Claude Code 里的 **Skill（技能模块）**。

它内置了一套完整的账户调研框架，覆盖：
- **WHAT**：账户定义、子类型、关键规则
- **WHY**：用户价值、平台价值、竞争机会点
- **HOW**：合规资质、实现路径、金融能力
- **WHEN & WHERE**：多市场横向对比
- **竞品分析**：结构化多维度竞品对比
- **产品设计启示**：将调研转化为可执行产品建议

你不需要手动复制任何提示词，**直接说你想调研什么就行**，Claude 会自动按框架执行。

**每次调研开始时，Claude 会：**
1. 确认调研深度和输出用途
2. 询问是否有额外的参考信息可以提供（如已有报告、竞品资料等）
3. 主动通过搜索获取最新的官方规则和竞品信息，而非仅依赖训练数据

## 基本用法：直接说你想做什么

### 生成完整调研报告

```
帮我调研新加坡的联名证券账户
```

```
帮我做澳大利亚退休账户的完整调研报告
```

```
帮我调研香港 MPF 账户，重点看竞品分析和产品设计启示
```

Claude 会输出包含以下内容的完整报告：

```
一、总括         ── 背景与调研范围
二、账户简介     ── 定义、子类型对比表、关键规则表（附来源）
三、多市场横向对比 ── 与其他市场同类制度对比
四、市场概览     ── 规模、增长趋势、用户结构
五、竞品分析     ── 能力/产品/费用/清结算横向对比表
六、监管合规     ── 监管框架、关键合规要点
七、产品设计启示 ── 定位、MVP、差异化、风险
```

---

### 只做某个阶段的调研

不需要完整报告时，可以指定阶段：

| 你想了解什么 | 示例指令 |
|-------------|---------|
| 账户基本概念和规则 | `帮我了解美国 IRA 账户的基本定义、子类型和关键规则` |
| 账户的用户和平台价值 | `分析联名账户对用户和券商平台分别有什么价值` |
| 合规资质和实现路径 | `调研在新加坡提供联名账户需要哪些牌照和合规要求` |
| 多市场横向对比 | `对比美国、澳大利亚、新加坡、香港四个市场的退休账户制度` |
| 竞品深度分析 | `帮我对比 Fidelity、Schwab、Robinhood 在退休账户上的竞品差异` |
| 产品设计启示 | `基于以上调研，给出对我们产品设计的启示和 MVP 建议` |

## 进阶用法

### 分阶段深挖（宽度 → 深度）

推荐工作流：

```
第一步  完整报告        →  建立整体认知，确认调研方向
第二步  追加 Phase 1-2  →  补充账户规则细节和价值分析
第三步  追加 Phase 5    →  深化竞品对比（可指定具体竞品清单）
第四步  追加 Phase 6    →  输出产品设计启示，对接内部需求
```

### 追问技巧

Claude 的初次输出是起点，可以继续在同一对话里追问：

```
请把竞品对比表加入 Interactive Brokers，按同样维度补充
```
```
帮我把产品启示部分改写成适合 PPT 的要点格式
```
```
数据来源只保留官方监管机构链接，去掉其他来源
```
```
请输出精简版本，每个章节不超过 300 字，重点突出结论
```

### 多市场并行对比

```
请同时对比新加坡、香港、澳大利亚三个市场的联名账户制度，
输出统一的横向对比表
```

## 如何修改 Skill 内容

Skill 的所有内容存放在一个文件里，可以随时修改：

**文件路径：**
```
C:\Users\jadezhang\.agents\skills\account-research\SKILL.md
```

**修改方式：**
- 用记事本或 VS Code 打开直接编辑
- 或者在 Claude Code 对话里说"帮我修改 account-research skill，把 XX 改成 XX"

修改后**不需要重启**，下次对话自动生效。

**常见修改场景：**

| 想修改什么 | 在文件里找什么位置 |
|-----------|-----------------|
| 增加新的调研维度 | 找对应 Phase，在输出要求里追加 |
| 调整竞品对比表的列 | 找 Phase 5，修改"对比维度"部分 |
| 改变默认输出语言 | 在"输出要求"末尾加语言说明 |
| 新增一个调研阶段 | 在文件末尾按同样格式添加 Phase 7 |

## 常见问题

**Q：Claude 输出的内容准确吗？**

Skill 会优先搜索官方监管机构、竞品官网、权威财经媒体等可信来源，不引用个人博客或无来源的论坛内容。对于可能过期的数据（如年度缴费上限、费率），输出中会标注 ⚠️ 提醒，建议通过附带的来源链接核实最新版本。

**Q：我有一些相关资料，可以提供给 Claude 参考吗？**

可以。调研开始时 Claude 会主动询问是否有额外的参考信息。你可以直接将相关内容粘贴到对话中，或在 Claude Code 里让 Claude 读取本地文件。你提供的参考信息优先级高于搜索结果，会被优先整合进输出。

---

**Q：输出是中文还是英文？**

默认跟随你提问的语言。想要指定语言，在问题后面加一句就行：
```
...请用中文输出
```

---

**Q：这个 Skill 和 claude.ai 网页版有什么区别？**

| | Claude Code Skill | claude.ai 网页版 |
|--|---|---|
| 触发方式 | 直接对话，自动加载 | 需要手动复制提示词 |
| 联网搜索 | 自动搜索官方和权威来源 | 有限支持 |
| 上下文保留 | 同一对话内完整保留 | 同一对话内完整保留 |
| 修改框架 | 编辑本地文件即可 | 需要每次手动修改提示词 |
| 适合场景 | 日常使用、持续迭代 | 临时使用、分享给他人 |

## 如何分享给同事

### 第一步：把 Skill 文件发给同事

将以下文件通过飞书/邮件/共享盘发送给同事：

```
account-research-skill.md（即 SKILL.md 的内容）
```

> 发这份 `.md` 文件就够了，不需要发其他东西。

---

### 第二步：同事安装 Skill（3 种方式）

#### 方式 A：让 Claude Code 帮忙安装（最简单，推荐）

同事只需要：

1. 打开 Claude Code
2. 把收到的 `account-research-skill.md` 文件内容完整复制
3. 发送以下内容给 Claude Code：

```
请帮我把以下内容安装为名为 account-research 的 Claude Code Skill：

（粘贴 SKILL.md 的全部内容）
```

Claude Code 会自动完成建目录、写文件、创建软链接的全部步骤。

---

#### 方式 B：手动安装（Windows，适合想自己操作的人）

打开 **Git Bash**（安装 Claude Code 时通常已附带），依次运行：

```bash
# 第一步：建目录
mkdir -p ~/.agents/skills/account-research

# 第二步：把收到的 SKILL.md 放进去
# 将文件复制到 C:\Users\你的用户名\.agents\skills\account-research\SKILL.md

# 第三步：创建软链接（让 Claude Code 能识别到）
ln -s ~/.agents/skills/account-research ~/.claude/skills/account-research
```

第二步也可以直接在文件资源管理器里操作：打开
`C:\Users\你的用户名\.agents\skills\account-research\`
把收到的文件放进去，改名为 `SKILL.md`。

---

#### 方式 C：直接使用网页版（无需安装）

如果同事只是偶尔用一次，不想安装，也可以直接在 [claude.ai](https://claude.ai) 网页版使用：

1. 打开 `account-research-skill.md`，找到**快速启动模板**部分
2. 复制代码块内容，把 `{{账户类型}}` 和 `{{目标市场}}` 替换为实际内容
3. 粘贴到 claude.ai 对话框发送

这种方式不需要安装任何东西，但每次都要手动复制提示词。

---

### 第三步：验证安装成功

安装完成后，同事在 Claude Code 里发送：

```
帮我调研新加坡的联名账户
```

如果 Claude 开始按 WHAT/WHY/HOW 框架输出结构化调研报告，说明安装成功。

---

### 三种方式对比

| | 方式 A（Claude 安装） | 方式 B（手动安装） | 方式 C（网页版） |
|--|--|--|--|
| 难度 | 最简单 | 中等 | 最简单 |
| 需要 Claude Code | 是 | 是 | 否 |
| 安装后使用 | 直接对话 | 直接对话 | 每次复制提示词 |
| 适合谁 | 大多数人 | 想了解原理的人 | 偶尔使用的人 |

### 后续更新 Skill

如果你修改了 Skill 内容，同事想同步更新，只需重新把最新的 `SKILL.md` 文件发给他们，让他们替换掉本地对应路径下的文件即可：

```
C:\Users\同事用户名\.agents\skills\account-research\SKILL.md
```

替换后不需要重启，下次对话自动生效。
