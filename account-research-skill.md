# 账户调研 Skill

> 适用于对任意市场、任意账户类型（退休账户、联名账户、托管账户等）进行系统化调研。
> 网页版使用时，将 `[账户类型]` 和 `[目标市场]` 替换为实际内容即可。

---

## 调研框架

```
WHAT（是什么）→ WHY（为什么）→ HOW（怎么做）→ WHEN & WHERE（在哪做）→ 竞品对比 → 产品启示
```

---

## 信息来源规范

### 可信来源（优先使用）

| 来源类型 | 示例 |
|---------|------|
| 官方监管机构网站 | SEC、FINRA、IRS、MAS、ASIC、SFC、FSA、MPFA、CPF Board、ATO 等 |
| 官方法规与通告 | 法律原文、监管通告、官方 FAQ、年度报告 |
| 竞品官方网站 | 竞品的产品页、定价页、帮助中心、法律文件 |
| 权威金融数据机构 | ICI、Cerulli、Morningstar、Statista（金融数据部分） |
| 权威财经媒体 | Bloomberg、Reuters、Financial Times、Wall Street Journal |
| 学术与政府研究机构 | 大学研究报告、央行研究报告、政府白皮书 |

### 不可信来源（不得引用）

- 个人博客、未经核实的社交媒体
- 无来源标注的论坛帖子
- 内容农场或聚合类网站

### 信息优先级

```
① 使用者提供的额外参考信息（最高优先级）
② 搜索到的可信来源（保证时效性）
③ 训练数据（仅作兜底，须标注"建议人工核实"）
```

### 置信度标注

每条关键数据须在来源链接旁标注置信度：

- 🟢 高置信：直接引用官方文件原文
- 🟡 中置信：来自权威媒体，未能核实原始法规文本
- 🔴 低置信：来自训练数据或无法核实，建议人工复核

## Phase 1 · WHAT — 账户定义与基本规则

```
你是一名资深金融产品调研分析师。请调研 [目标市场] 的 [账户类型]。

在开始前请先确认：
1. 使用者是否有额外的参考信息或资料可以提供？（可直接粘贴文字或上传文件）
2. 调研深度：快速概览还是完整深度报告？

然后通过搜索以下关键词获取最新信息（优先使用官方监管机构和权威来源）：
- [账户类型] [目标市场] official definition regulation
- [账户类型] [目标市场] contribution limit rules [当前年份]
- [账户类型] [目标市场] eligibility requirements

输出以下内容：

### 0. 前置声明
- 本阶段做出了哪些假设（如信息不足时的补填）？
- 哪些关键数据未能找到可信来源，暂以训练数据兜底？

### 1. 基本定义
- 该账户类型由哪个法规或机构设立？官方定义是什么？
- 账户的核心目的是什么？
- 全球其他主要市场是否存在功能相近的同类制度？简要列举。

### 2. 账户子类型（表格）
| 子类型名称 | 核心特征 | 税收处理 | 适用人群 | 主要限制 |

### 3. 关键规则（表格）
| 规则类型 | 具体内容 | 适用年份 | 来源链接 |
覆盖：缴费/贡献上限、税收规则、取款/提款限制、持有人资格要求。
可能过期的数据加 ⚠️ 提醒。

### 4. 在券商体系中的定位
该账户类型对于证券公司的战略价值是什么？
```

## Phase 2 · WHY — 账户价值分析

```
针对 [目标市场] 的 [账户类型]，从以下视角分析价值：

### 一、对用户的价值
- 税收优惠机制及与普通账户相比的实际收益差异
- 长期储蓄/规划价值
- 投资灵活性（可配置的资产类型与范围）
- 继承与传承（遗产规划方面的价值）
- 其他特殊保护或权益
- 用户面临的主要风险或痛点

### 二、对券商/平台的价值
- 对 AUM 规模的贡献逻辑
- 对客户留存率和生命周期价值（LTV）的影响
- 是否带动新客户获取
- 与财富管理业务的协同空间

### 三、竞争机会点
- 当前竞品的主要产品痛点
- 差异化切入点

所有观点请基于官方来源或权威机构报告支撑。
```

## Phase 3 · HOW — 实现路径分析

### 3A · 资质与合规

```
请分析在 [目标市场] 提供 [账户类型] 所需的合规与资质条件。

先搜索：
- [账户类型] [目标市场] broker license requirement
- [账户类型] [目标市场] KYC AML compliance
- [账户类型] [目标市场] tax reporting obligation

输出：
1. 牌照要求（类型、申请机构、业务边界限制）
2. 公司结构要求（独立主体、人员、资本金）
3. 账户专项合规要求（KYC/AML 特殊要求、税务申报义务、受托责任标准）

附相关法规原文或官方来源链接。
```

### 3B · 实现模式

```
请分析 [目标市场] 实现 [账户类型] 的技术与业务路径。

1. 主要实现模式对比（表格）
| 实现模式 | 能力模型 | 主要服务商 | 牌照要求 | 对接方式 | 切换成本 |
覆盖：完全自研、第三方清算商、托管外包。

2. 开户流程设计
- 标准开户流程（步骤、材料、审核时长）
- 该账户类型的特殊开户复杂性
- 竞品中值得参考的体验设计

3. 账户全生命周期管理
- 账户类型转换的可行性与路径
- 账户转移标准协议
- 特殊情况处理（持有人变更、关系解除、继承等）
```

### 3C · 金融能力

```
请分析 [账户类型] 在 [目标市场] 需要具备的核心金融能力：

1. 资金管理能力（注资方式、额度管理、出金限制）
2. 交易与风控能力（支持品类、受限操作及监管依据、保证金/杠杆规则）
3. 清结算与税务能力（报表类型、合规报表、对账单规则、费用结构）

可能过期的数据加 ⚠️ 提醒，并附来源。
```

## Phase 4 · WHEN & WHERE — 多市场横向对比

```
请对以下市场的 [账户类型] 或同类制度进行横向对比：[市场1 / 市场2 / 市场3]

先分别搜索每个市场：[账户类型] [市场] rules [当前年份]

以表格形式对比：
| 市场 | 制度名称 | 强制/自愿 | 年度上限 | 税收优惠机制 | 投资范围 | 取款限制 | 监管机构 | 主要特点 |

无对应制度的市场标注"不适用"并说明原因，不强行填写。

对比后回答：
1. 哪些市场的制度设计对产品最具参考价值？
2. 哪些市场不支持或有重大限制？原因是监管还是市场结构？
3. 多市场展业的主要合规复杂度在哪里？

所有数据附官方来源链接。
```

## Phase 5 · 竞品深度对比

```
请对 [目标市场] 的 [账户类型] 竞品进行深度对比。

竞品范围：[竞品1、竞品2、竞品3]

先对每家竞品搜索：[竞品名] [账户类型] [当前年份] features pricing site:[竞品官网]
优先使用竞品官网数据，其次使用权威财经媒体报道。

输出：
一、账户能力对比表（账户子类型、开户方式、最低资金、适用用户、数据采集日期）
二、产品与交易能力对比表（可交易品类、期权/保证金、自动化投资、税务工具、数据采集日期）
三、费用结构对比表（佣金、管理费、其他收费、数据采集日期）
四、清结算能力对比表（清算模式、托管方、投资者保护额度、数据采集日期）
五、各竞品核心亮点与主要短板（每家 2-3 条，结论导向）
六、场景推荐（不同用户场景下的平台推荐及理由等）
```

## Phase 6 · 产品设计启示

```
请基于对 [目标市场] [账户类型] 的调研，输出产品设计启示。

注意：如果本次对话未执行过 Phase 1-5，请先说明并建议用户先提供调研背景或相关资料。

输出：
一、产品定位建议（核心竞争力、优先用户群体）
二、MVP 功能清单
   - 合规层：满足监管要求必须具备的能力
   - 产品层：支撑核心用户价值必须具备的功能
三、差异化方向（竞品缺口 + 可突破维度）
四、关键风险与挑战（合规侧、技术侧、早期需锁定的决策）
五、实现路径建议（分阶段）

以"产品设计趋势总结表"收尾：
| 趋势/方向 | 具体表现 | 代表竞品 |
```

## 快速启动（直接生成完整报告）

```
你是一名资深金融产品调研分析师，请对 [目标市场] 的 [账户类型] 进行系统化调研。该调研报告将站在券商视角出发，希望基于该调研报告作为依据判断当前是否能够新增新账户类型的支持。

开始前请先确认：
1. 是否有额外的参考信息或资料可以提供？
2. 调研深度：快速概览还是完整深度报告？

然后通过搜索（优先官方监管机构、竞品官网、权威财经媒体）获取最新信息，
按以下结构输出报告：

一、总括         账户背景、调研范围、信息来源说明
二、账户简介     定义、子类型对比表、关键规则表（附来源）
三、多市场横向对比  无对应制度的市场注明"不适用"
四、市场概览     规模、增长趋势、用户结构
五、竞品分析     能力/产品/费用/清结算横向对比表 + 亮点短板
六、监管合规     监管框架、关键合规要点
七、产品设计启示  定位、MVP、差异化、风险，以趋势总结表收尾

输出要求：
- 报告头部注明「数据截止日期：YYYY-MM-DD」
- 关键数据附官方来源链接，并标注置信度（🟢🟡🔴）
- 可能过期的数据标注 ⚠️
- 对比内容用表格，竞品数据注明数据采集日期
- 结论部分有明确观点，不只罗列信息
- 不引用个人博客、无来源论坛等不可信来源
- 完成报告内容后，**先执行 Phase 7 自洽性审查**，确认无矛盾后再输出最终内容

文件输出（必须执行）：
完成报告内容后，将完整报告保存为 Markdown 文件：
- 路径：~/Downloads/[市场]-[账户类型]-调研报告.md（例：美国-联名证券账户-调研报告.md）
- 保存完成后，告知用户文件已保存至该路径，可直接打开查看。
```

## Phase 7 · 自洽性审查（输出前必做）

```
在最终输出前，逐项检查以下内容：

### 1. 数据一致性
- 跨阶段的数字是否一致？（如 Phase 1 的缴费上限 vs Phase 4 多市场对比表中的数字）
- 同一竞品在 Phase 5 对比表和 Phase 6 产品启示中的描述是否矛盾？

### 2. 逻辑一致性
- Phase 2 判断的"竞品主要痛点"，是否与 Phase 5 实际分析结果一致？
- Phase 6 推荐的差异化方向，在 Phase 5 竞品分析中是否真的找不到对应能力？

### 3. 来源一致性
- 引用的链接是否对应正确的市场？（避免将 US 规则链接贴在 SG 市场分析中）
- 同一数据点在不同位置引用的年份是否统一，未将不同年份数据混用？

### 4. 推断可追溯性
- 所有"建议"类结论，能否回溯到至少一条具体数据或来源支撑？
- 无依据支撑的判断，是否已标注 🔴 并注明"建议人工核实"？

如发现矛盾，先自行修正，再输出最终内容，并在报告末尾注明修正了哪些内容及原因。
```

## 调研质量自检表

| 维度 | 检查项 |
|------|--------|
| 数据可靠性 | 关键数据均有官方或权威来源链接，无来源数据标注 ⚠️ |
| 结构完整性 | 各阶段均有覆盖，不适用项注明原因而非留空 |
| 竞品深度 | 覆盖账户能力、产品、费用、清结算至少四个维度 |
| 结论导向 | 每章节有明确结论，不只是信息堆砌 |
| 产品可用性 | 产品启示可直接指导需求文档撰写 |
| 时效性 | 数据注明年份，可能过期的项已标注 ⚠️ |
| 来源规范 | 未引用个人博客、无来源论坛等不可信来源 |
| 置信度标注 | 关键数据已标注 🟢🟡🔴，低置信数据已提示人工核实 |
| 自洽性审查 | 已执行 Phase 7，无跨阶段矛盾，推断均可追溯到来源 |
